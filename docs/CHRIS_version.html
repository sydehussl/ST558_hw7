Making Recipes

Our first recipe will create a “weekend” true/false flag to use in our model. We also normalize the numeric predictors and create dummy variables for our categorical predictors.

rec_1 <- 
  recipe(bike_count ~ ., data = bike_rentals) |>
  update_role(date, new_role = "ID") |>
  step_date(date, features = "dow") |>
  step_mutate(weekend = factor(if_else(date_dow %in% c("Sat", "Sun"),
                                        TRUE, FALSE)),
              keep_original_cols = FALSE) |>
  step_normalize(all_numeric_predictors()) |>
  step_dummy(all_nominal_predictors()) |>
  step_zv(all_predictors())

In recipe 2, we add interaction terms to the previous recipe:

rec_2 <-
  rec_1 |>
  step_interact(terms = ~ starts_with("seasons"):starts_with("holiday")) |>
  step_interact(terms = ~ starts_with("seasons"):temp) |>
  step_interact(terms = ~ rainfall:temp)

Finally in recipe 3, we add quadratic (second-order) terms to the model:

rec_3 <-
  rec_2 |>
  step_mutate(rainfall_sq = rainfall ^ 2,
              snowfall_sq = snowfall ^ 2,
              temp_sq = temp ^ 2,
              humidity_sq = humidity ^ 2,
              wind_sq = wind ^2,
              visibility_sq = visibility ^ 2,
              dew_point_sq = dew_point ^ 2,
              solar_rad_sq = solar_rad ^ 2)

Create and Fit Models

We’re ready to fit the models! First, we need to specify the type and engine for the model. After that, we create the workflows using the recipes we created earlier.
